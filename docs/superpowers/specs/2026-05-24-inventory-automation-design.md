# 盤點表自動化工具 — 設計規格

**日期：** 2026-05-24  
**狀態：** 待審核（v4 修訂 — 已對照智生活欄位標題）  
**專案：** Toolbox / frontend

---

## 1. 背景與目標

### 現有工作流程
1. 到智生活複製資料
2. 貼到 Google Sheets
3. 手動調整欄寬
4. 輸出 PDF

### 目標
使用者在工具頁面上，依類別**分別貼上**智生活複製的資料，自動：
- 解析各區塊內容
- 將地址轉換為戶別代號（`8號11樓之7` → `11A7`；`(店8)` → `店8`）
- 整理成固定 **6 欄**格式
- 合併輸出 PDF（**空區塊略過**）

### v3 核心變更
> **不再**一次貼上全部資料後自動分區。改由頁面上 **8 個獨立貼上區塊** 擔任第一次分類；使用者從智生活分次複製、貼入對應區塊。

### 範圍外（v1 不做）
- 後端 API、資料持久化
- 儲存格手動編輯
- Excel 匯出
- 自動去重、日期篩選
- 跨區塊自動搬移（貼錯區不自動修正）

## 2. 貼上區塊定義

頁面固定 **8 個貼上區**，順序即 PDF 輸出順序：

| ID | 區塊名稱 | 智生活複製來源 | 解析器 | 空則 |
|----|----------|----------------|--------|------|
| `mail` | 一般掛號 | 掛號列表 | 單行 Tab | 略過 |
| `pkg-a` | A棟包裹 | 8號包裹 | 單行 Tab | 略過 |
| `pkg-b` | B棟包裹 | 10號包裹 | 單行 Tab | 略過 |
| `pkg-shop` | 店面包裹 | 店面包裹 | 單行 Tab | 略過 |
| `cash-a` | A棟現金 | 8號寄放現金 | 多行代管 | 略過 |
| `cash-b` | B棟現金 | 10號寄放現金 | 多行代管 | 略過 |
| `cash-shop` | 店面現金 | 店面寄放現金 | 多行代管 | 略過 |
| `custody-other` | 寄放非現金 | 非現金寄放 | 多行代管 | 略過 |

## 3. 原始資料格式（智生活欄位標題）

複製時常含**標題列**，解析器須自動略過（見 §3.3）。

### 3.1 一般掛號 / 包裹（單行、Tab 分隔）

適用區塊：`mail`、`pkg-a`、`pkg-b`、`pkg-shop`

**智生活標題列：**

| 欄序 | 欄名 |
|------|------|
| 1 | `#` |
| 2 | `戶別` |
| 3 | `包裹類型` |
| 4 | `收件人` |
| 5 | `包裹條碼後七碼` |
| 6 | `貨到時間` |
| 7 | `備註` |

範例：
```
#	戶別	包裹類型	收件人	包裹條碼後七碼	貨到時間	備註
4	8號12樓之6	一般掛號	廖O俊	0501632	05/21	【位置】櫃檯
12	8號5樓之3	包裹	蕭O文	6300790	05/23	X3【位置】宅配室
```

### 3.2 寄放現金 / 寄放非現金（多行儲存格）

適用區塊：`cash-a`、`cash-b`、`cash-shop`、`custody-other`

**智生活標題列：**

| 欄序 | 欄名 |
|------|------|
| 1 | `#` |
| 2 | `寄放人` |
| 3 | `寄放類型` |
| 4 | `領取人` |
| 5 | `寄放時間` |
| 6 | `位置` |
| 7 | `備註` |

範例：
```
#	寄放人	寄放類型	領取人	寄放時間	位置	備註
B30	8號10樓之1
【黃O幃】	現金(30)	【貨款】	2026/05/23 11:52	櫃檯抽屜(錢袋)	-170
```

`寄放人`、`領取人` 欄位內可含換行（地址與 `【姓名】` 分行）。

### 3.3 標題列偵測

略過符合以下條件的行：

```typescript
cells[0] === '#' && (cells[1] === '戶別' || cells[1] === '寄放人')
```

### 3.4 測試 Fixture

| 檔案 | 用途 |
|------|------|
| `__fixtures__/sample-mail.txt` | 一般掛號 |
| `__fixtures__/sample-package.txt` | A 棟包裹 |
| `__fixtures__/sample-cash.txt` | 寄放現金 |
| `__fixtures__/sample-non-cash.txt` | 寄放非現金 |

## 4. 戶別代號轉換規則

### 4.1 住宅（A / B 棟）

```
8號{樓層}樓之{之幾}   →  {樓層}A{之幾}
10號{樓層}樓之{之幾}  →  {樓層}B{之幾}
```

### 4.2 店面

```
(店N)…  →  店N
```

### 4.3 其他
- `管理室`：保留原文
- 無法辨識：保留原文，UI 琥珀色提示

> v3 不再依地址自動決定貼上區塊；轉換僅用於「收件戶別」欄顯示。

## 5. 輸出表格欄位（統一 6 欄）

| # | 欄名 | 說明 |
|---|------|------|
| 1 | ☐ | 空白勾選框 |
| 2 | 編號 | 見 §5.1 |
| 3 | 收件戶別 | 代號轉換（§4） |
| 4 | 收件人 | |
| 5 | 登記時間 | |
| 6 | 位置 | 移除 `【位置】`、`X{n}` 前綴 |

### 5.1 欄位映射

**掛號 / 包裹區塊：**

| 輸出欄 | 智生活欄 | 說明 |
|--------|----------|------|
| ☐ | — | 空白 |
| 編號 | `#` | 如 `4`、`12` |
| 收件戶別 | `戶別` | → 代號（§4） |
| 收件人 | `收件人` | |
| 登記時間 | `貨到時間` | 如 `05/21` |
| 位置 | `備註` | 移除 `【位置】`、`X{n}` 前綴 |

**不輸出：** `包裹類型`（已由貼上區塊區分）、`包裹條碼後七碼`。

**現金 / 非現金區塊：**

| 輸出欄 | 智生活欄 | 說明 |
|--------|----------|------|
| ☐ | — | 空白 |
| 編號 | `#` | 如 `B30` |
| 收件戶別 | `寄放人` 內地址 | → 代號；見 §5.2 |
| 收件人 | `寄放人` 內 `【姓名】` | 見 §5.2 |
| 登記時間 | `寄放時間` | |
| 位置 | `位置` | 如 `櫃檯`、`宅配室` |

**不輸出：** `寄放類型`、`領取人`、`備註`（v1）。

### 5.2 寄放人欄位解析

從 `寄放人`（及必要時 `領取人`）擷取地址與姓名：

1. 將 `寄放人` 依換行拆行
2. 符合 `8號…` / `10號…` / `(店N)…` / `管理室` → **地址** → 收件戶別代號
3. 符合 `【…】` → **收件人**
4. 若 `寄放人` 無地址（如 `B38` 空白），改從 `領取人` 欄取地址／姓名
5. 若僅有類型無姓名（如 `B84` 第二行 `非現金(鑰匙)`），`領取人` 的 `【訪客】` 可作收件人

**範例：**

| 原始 | 編號 | 收件戶別 | 收件人 |
|------|------|----------|--------|
| `B30` + `8號10樓之1` + `【黃O幃】` | B30 | 10A1 | 黃O幃 |
| `B38` 空白 + 領取人含 `10號7樓之7` | B38 | 7B7 | 全戶O知 |
| `B10` + `管理室` | B10 | 管理室 | （江O勳 在領取人欄，可擷取） |

## 6. 解析器架構（v3 簡化）

```
parseAllZones(zones: Record<ZoneId, string>)
  ├── for each zone with non-empty content:
  │     ├── pickParser(zoneId)     // mailPackage | custody
  │     ├── parseZone(raw)
  │     ├── mapToOutputRows()      // → 6 欄
  │     └── attach sectionTitle    // 區塊名稱
  └── return PdfSection[]          // 僅含非空區
```

### 6.1 模組職責

| 檔案 | 職責 |
|------|------|
| `zones.ts` | 8 區塊定義、ID、標題、解析器類型 |
| `parseMailPackage.ts` | 單行 Tab 解析 |
| `parseCustody.ts` | 多行 B-record 狀態機 |
| `convertUnitCode.ts` | 地址 → 戶別代號 |
| `mapToOutputTable.ts` | → 6 欄 |
| `parseAllZones.ts` | 統一入口 |
| `exportInventoryPdf.ts` | PDF 匯出 |

### 6.2 移除的模組（v2 → v3）
- ~~`splitSections.ts`~~ — 使用者手動分區

## 7. UI 設計

### 7.1 路由
- `/` — 工具列表
- `/tools/inventory-automation` — 盤點表工具

### 7.2 頁面結構

```
ToolPageShell
├── WorkflowGuide（複製 → 分區貼上 → 匯出）
├── PreviewSettings（報表標題、PDF 方向）
├── ZonePasteGrid（8 個貼上區，可折疊）
│   ├── ZoneCard × 8
│   │   ├── 標題 + 列數 badge（解析後）
│   │   ├── textarea（貼上區）
│   │   ├── 「清除」按鈕
│   │   └── 折疊預覽表（解析後顯示）
├── GlobalActions（全部解析 / 全部清除）
└── ExportBar（下載 PDF，至少一區有資料才可按）
```

### 7.3 ZoneCard 行為
- **貼上時**：可選自動解析該區，或按「全部解析」一次處理
- **空區**：不顯示於 PDF；預覽區顯示「尚未貼上資料」
- **有資料**：顯示列數，展開可預覽 6 欄表格

### 7.4 版面建議
- 桌面：2 欄 grid（4×2 或 8 列單欄，依可讀性）
- 行動：單欄堆疊
- 各區 textarea 高度：`min-h-24`，有內容可自動增高

### 7.5 PDF 設定
- 標題：預設「盤點表」
- 方向：預設橫向 A4
- 檔名：`盤點表-YYYY-MM-DD.pdf`

## 8. PDF 輸出格式

僅輸出**有貼上內容**的區塊，順序固定：

```
1. 一般掛號
2. A棟包裹
3. B棟包裹
4. 店面包裹
5. A棟現金
6. B棟現金
7. 店面現金
8. 寄放非現金
```

每區格式：
```
【{區塊名稱}】
☐ │ 編號 │ 戶別 │ 收件人 │ 登記時間 │ 位置
...
```

- 字體：Noto Sans TC
- 勾選欄：窄欄置中 `□`
